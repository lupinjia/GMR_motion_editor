## 1. 项目概述

### 1.1 目标
为GMR (General Motion Retargeting)项目开发一个图形化界面工具，用于可视化、编辑和导出机器人运动数据。

### 1.2 核心功能
- **导入/导出**: 加载和保存GMR格式(.pkl)的机器人运动数据
- **可视化**: 使用MuJoCo实时显示机器人运动
- **波形显示**: 独立窗口显示各数据维度的波形曲线，支持多维度数据同步查看
- **剪辑**: 简单的起止时间裁剪功能
- **多机器人支持**: 支持项目中所有17种机器人模型

## 3. 数据流设计

### 3.1 播放循环
```
1. QTimer触发 (间隔 = 1000ms/FPS)
2. 获取当前帧数据 (root_pos, root_rot, dof_pos)
3. 发送到MuJoCo viewer显示
4. 更新TimelineWidget当前位置
5. 到达裁剪终点时暂停或循环
```

### 3.2 裁剪操作
```
1. 用户调整起点/终点手柄
2. TimelineWidget发出信号
3. MainWindow更新裁剪范围
4. 导出时，GMRDataManager截取[start:end]区间数据
```

### 3.3 波形显示同步
```
1. 用户打开Waveform Display窗口 (Ctrl+W)
2. 从下拉框选择要显示的数据key
3. 每个维度显示为独立行，包含波形曲线
4. 播放时，黄色竖线在各波形中同步移动
5. 时间轴拖拽时，波形窗口实时更新当前帧位置
```

### 3.4 数据格式
- 复用现有pickle格式，支持所有可用字段：
  - `fps`: 帧率
  - `root_pos`: 根节点位置 [N, 3]
  - `root_rot`: 根节点旋转 [N, 4] (xyzw格式)
  - `dof_pos`: 关节位置 [N, n_dof]
  - `root_lin_vel`: 根节点线速度 [N, 3]
  - `root_ang_vel`: 根节点角速度 [N, 3]
  - `root_euler`: 根节点欧拉角 [N, 3]
  - `dof_vel`: 关节速度 [N, n_dof]
  - `key_body_pos_relative_to_base`: 关键点相对位置 [N, bodies, 3]
  - `local_body_pos`: 局部身体位置
  - `link_body_list`: 链接身体列表
- 裁剪后保持原有字段不变，只截取帧范围
- 波形显示支持所有数值型数组数据（1D、2D、3D）

## 4. 界面设计

### 4.1 主窗口规格
- **尺寸:** 800x600px
- **技术栈:** PyQt6
- **3D视图:** 独立MuJoCo窗口

### 4.2 菜单栏
```
File:
  - Open (Ctrl+O)
  - Save (Ctrl+S)
  - Save As
  - Export Clip
  - Exit

Edit:
  - Undo
  - Redo
  - Reset View

View:
  - Toggle Loop
  - Reset Timeline
  - Waveform Display (Ctrl+W)  # 打开波形显示窗口

Help:
  - About
```

### 4.3 时间轴控件
- **进度条:** 显示总帧数（如 0 - 120）
- **起点手柄:** 蓝色，可拖动
- **终点手柄:** 红色，可拖动
- **当前帧指示器:** 黄色竖线，随播放移动
- **数字输入框:** 精确设置起止帧

### 4.4 控制面板
- 播放/暂停按钮
- 速度滑块 (0.5x - 2.0x)
- 循环开关
- 当前帧/总帧数显示
- 时间显示 (秒)

### 4.5 波形显示窗口 (Waveform Display)
- **窗口特性:** 独立置顶窗口，不遮挡主窗口
- **数据选择:** 下拉框选择要显示的数据key
- **多维度显示:** 每个维度一行，显示完整波形曲线
- **信息显示:** 每行左上角显示维度名称、当前值(Cur)、最小值(Min)、最大值(Max)
- **当前帧指示:** 黄色竖线在波形中实时指示当前帧位置
- **支持数据类型:** 所有numpy数组数据（1D、2D、3D）
- **异常处理:** None值、空数组、标量值均有对应显示

### 4.6 状态栏
- **左侧:** 当前状态（就绪/播放中/裁剪中）
- **右侧:** 文件路径、机器人名称、总帧数

## 5. 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 文件不存在/损坏 | QMessageBox提示，恢复到之前状态 |
| 机器人模型缺失 | 提示下载，提供安装指南链接 |
| 裁剪范围无效 | 自动修正（起点>终点时交换），提示用户 |
| MuJoCo窗口关闭 | 自动停止播放，重新打开时恢复 |
| 波形数据为None | 显示"No data available (None)"提示 |
| 波形数据类型不支持 | 显示具体类型信息，不绘制波形 |
| 内存不足 | 分段加载，显示加载进度 |

## 6. 技术选型

### 6.1 GUI框架: PyQt6
- **选择理由:** 
  - 与现有GMR项目兼容（已有PyQt6可视化代码）
  - 成熟稳定，生态丰富
  - 支持自定义控件

### 6.2 3D渲染: MuJoCo
- **选择理由:**
  - 项目已有成熟的MuJoCo集成
  - RobotMotionViewer可直接复用
  - 机器人渲染效果最佳

## 10. 用户操作流程

### 10.1 基本使用
1. 运行 `python scripts/gmr_visualizer.py`
2. File → Open 选择 .pkl 文件
3. 选择对应的机器人类型
4. 点击播放按钮查看运动
5. 调整时间轴手柄设置裁剪范围
6. File → Export Clip 导出裁剪后的数据

### 10.2 快捷键
- `Ctrl+O`: 打开文件
- `Ctrl+S`: 保存
- `Ctrl+Shift+S`: 另存为
- `Ctrl+W`: 打开波形显示窗口
- `Space`: 播放/暂停
- `←/→`: 上一帧/下一帧
- `Home`: 跳到裁剪范围开始
- `End`: 跳到裁剪范围结束
